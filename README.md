### Project description
As a security analyst at an international organization, who primarily works with a research team, I need to ensure that the research team members have permissions only necessary for their duties. The principle of the least privilege decreases the attack surface and, consequently, improves organization's security.

In order to ensure that permissions correspond to provided authorisation, I have to examine permissions in the file directory. In case of any disparities, I will need to amend those by using authorisation commands in a Linux bash.

### Summary
First, I executed `ls` command to check for current files, directories, and permissions aligned to them. Then, I described the permissions string and what each symbol in the string stands for and relates to. After that, I changed file permissions and elaborated on the `chmod` command I used to achieve this. Finally, I removed all superfluous permissions for drafts directory, and at the end reviewed permissions with the `ls -la` command so as to ensure that I executed previous steps properly.
